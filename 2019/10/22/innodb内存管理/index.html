Innodb内存管理

发表于 2019-10-22 分类于 数据库 Disqus：

源码文件：mem0pool.c
功能：为innoDB中的内存堆对象提供所需的内存空间

内存池的元信息

结构体mem_pool_struct记录了内存池的元信息，而结构体mem_area_t则记录了内存块的元信息

源码分析

内存池的核心在于内存管理，因此这里主要分析三点：1. 内存池的创建，2. 内存分配，3. 内存回收
由于内存池会被多个线程使用，因此，在内存池的相关调用函数会出现加锁和解锁的操作，其中加解锁函数如下：

1.创建通用内存池

2.向通用内存池请求内存

3.释放内存块
释放内存块时，除了将被释放的内存块挂入相应链表，还会检测相邻的内存块是否可以合并。假设被释放的内存块大小为n，起始地址为s，内存池大小为m，合并规则是：

  1. 计算t = (m - s) / n % 2
  2. 若t = 1，设变量 q = -n和w = s - n, 否则设 q = n 和 w = s
  3. 检查地址s + q对应的内存块，
    1. 若大小等于被释放的内存块，且free=true，则可以合并，合并后，设n=2*n，s= w，跳转到步骤1
    2. 否则,结束释放程序

其中，步骤1和2实际是为了计算内存块的『伙伴』，在函数mem_area_get_buddy中实现，mem_area_get_buddy(mem_area_t, ulint, mem_pool_t)，参数依次为：内存块地址，内存块大小，内存池地址，输出为该内存块的『伙伴』
